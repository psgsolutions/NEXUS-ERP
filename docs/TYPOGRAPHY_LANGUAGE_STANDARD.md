## 1. 🏗️ สถาปัตยกรรมตัวอักษร (Typography Architecture)

| ระดับ (Level) | ขนาด (Size) | น้ำหนัก (Weight) | กรณีใช้งาน (Use Case) |
| :--- | :--- | :--- | :--- |
| **Header 1** | 20px - 22px | Black (900) | หัวข้อหน้าหลัก (Hub Title) |
| **Header 2** | 16px - 18px | Black (900) | หัวข้อกลุ่มข้อมูล (Section Header) |
| **Body (Normal)** | 14px | Medium (500) | ข้อมูลทั่วไป, ข้อความใน Input |
| **Body (Strong)** | 14px | Bold (700) | ชื่อเฉพาะ, หัวข้อตาราง |
| **Labels** | 11px | Black (900) | ป้ายกำกับช่องกรอก (Uppercase Only) |
| **Numeric** | 14px - 16px | Black (900) | ตัวเลขเงิน, รหัสเอกสาร (ใช้ Monospace) |

### 🛠️ กฎเหล็กการแสดงผล (Strict Rules)
- **[TYP-01] Non-italic Policy**: ห้ามใช้ตัวเอียง (Italic) ในทุกจุดของระบบ 100% เพื่อความชัดเจน
- **[TYP-02] Normal Tracking**: บังคับใช้ `tracking-normal` (ระยะห่างปกติ) ห้ามบีบหรือขยายช่องว่าง
- **[TYP-03] Vertical Alignment**: ตัวเลขต้องตั้งตรงและอ่านง่ายในรูปแบบ Monospace

## 2. 🇹🇭 มาตรฐานภาษาไทย (Thai Communication Protocol)

เน้นการใช้ภาษาที่ **"กระชับ-สุภาพ-เข้าใจง่าย"** แบบมืออาชีพ

| ภาษาเดิม (Technical) | ภาษาที่ใช้ในระบบ (Simple Thai) | คำอธิบาย |
| :--- | :--- | :--- |
| Inventory / Stock | **คลังพัสดุ / สินค้า** | ใช้คำว่า "พัสดุ" สำหรับงานโครงการ และ "สินค้า" สำหรับงานขาย |
| General Ledger | **สมุดรายวันกลาง** | ภาษาบัญชีที่เข้าใจง่ายสำหรับนักบริหาร |
| Internal Issue | **การเบิกพัสดุหน้างาน** | สื่อถึงการนำของออกจากคลังไปใช้ในโปรเจกต์ |
| Warehouse | **พิกัดคลัง / คลังสินค้า** | ใช้คำว่า "พิกัด" เพื่อสื่อถึงตำแหน่งจัดเก็บที่แน่นอน |
| Credit Term | **เครดิตเทอม / กำหนดจ่าย** | ใช้ทับศัพท์ที่นักธุรกิจคุ้นเคย |
| Soft Delete | **ถอนข้อมูล** | สื่อถึงการยกเลิกการแสดงผลแต่ข้อมูลยังอยู่ในสารบบ |
| Hard Delete (Purge) | **ลบข้อมูลถาวร** | คำสั่งเด็ดขาดสำหรับ Admin เท่านั้น |

## 3. 📖 พจนานุกรมศัพท์เฉพาะ (System Glossary)

- **SKU (Stock Keeping Unit)**: รหัสพิกัดพัสดุ (เช่น SKU-GEN-0001)
- **ชุดงาน (Kit / BOM)**: รายการวัสดุอุปกรณ์ที่จัดรวมกลุ่มไว้สำหรับการติดตั้ง 1 งาน
- **ผู้รับเหมาช่วง (Sub-contractor)**: พาร์ทเนอร์ช่างภายนอกที่รับงานติดตั้ง
- **ใบสั่งขาย (Sales Order)**: เอกสารยืนยันการขายและการจองพัสดุ
- **ใบวางบิล (Invoice)**: เอกสารเรียกเก็บเงินลูกค้า
- **การตรวจนับ (Audit)**: กระบวนการตรวจสอบความถูกต้องของพัสดุจริงเทียบกับระบบ
- **นิติบุคคล (Company/Node)**: บริษัทปฏิบัติการที่ใช้งานในระบบ
